Functions
       
create_origami_rectangle(num_helices: int, num_cols: int, twist_correction_deletion_spacing: int = 0, num_flanking_columns: int = 1, nick_pattern: origami_rectangle.NickPattern = <NickPattern.staggered: 1>) -> scadnano.DNADesign
Creates a DNA origami rectangle with a given number of helices and
"columns" (16-base-wide region in each helix). The columns include
the 16-base regions on the end where potential "edge staples" go,
as well as the two-column-wide "seam" region in the middle.
 
Here is a diagram of the sort of staples created for num_helices=6, num_cols=12. Helix indices are
labelled H0, H1, ... and column indices are labeled C0, C1, ... Each single symbol such as
``-``, ``+``, ``<``, ``+``, represents two DNA bases.
 
``.       C0    C1  C2 ...``
 
``. H0 <------+ +``
 
``.           | |``
 
``. H1 [------+ +``
 
Prints warning if number of bases exceeds 7249 (length of M13 scaffold),
but does not otherwise cause an error.
 
`num_cols` must be even.
 
If `twist_correction_deletion_spacing > 0`, adds deletions between crossovers in one out of
every `twist_correction_deletion_spacing` columns. (TODO: cite Sungwook's paper)
 
`num_flanking_columns` is the number of empty columns on the helix on each side of the origami.
 
`nick_pattern` describes whether nicks between staples should be "staggered" or not.
If `nick_pattern == origami_rectangle.NickPattern.staggered`, then a nick appears in a given helix
and column if the parity of the helix and column match (both even or both odd).
If `nick_pattern == origami_rectangle.NickPattern.unstaggered`, then a nick appears in only
even helices, but in every column. TODO: check this claim against the design
